## Описание сущностей
Document (Документ) - основная сущность, которая записывается в отдельный файл в корневой папке проекта и содержит в себе описание строительной площадки и всех строений на ней.

Element (Элемент) - базовый контейнер для всех элементов модели, который определяет имя элемента, его id и ссылку на стиль.

Style (Стиль) - сущность содержащая "префаб" элемента определенной категории, то есть заданные свойства и входящие элементы. Основное назначение - это описание одинаковых элементов модели в одном месте, для возможности наследования определения из этого описания. То есть если в проекте повторяется пространство квартиры, с одинаковыми разделениями на помещение, от пространство можно описать в стиле и ссылать на него из этих повторяющихся элементов пространств. Это позволяет задать описание в одном месте, и изменять сразу все повторяющиеся элементы, изменяя только это описание. 

DividingPlane (Разделяющая плоскость) - плоскость разделяющая пространство по горизонтали. Такая плоскость задается точкой вставки и отметкой (вертикальной координатой) в локальной системе пространства.

DividingRegion (Разделяющий регион) - часть разделяющей плоскости, ограниченная замкнутым контуром. Задается контуром и отметкой относительно разделяющей плоскости (==возможно отметкой относительно пространства==). На разделяющей плоскости могут быть несколько разделяющих регионов, их контуры не могу пересекаться, но могу быть вложены друг в друга. Если в ходе преобразований часть региона выходит за границы разделяющей плоскости, то эта часть "отрезается", то-есть не учитывается при построении, но не пропадает из описания.

DividingBoundary (Разделяющая граница) - линия разделяющая пространство по вертикали. Задается набором CheckPoint, которые задают отрезки разделяющая границы. То, как строится разделяющая граница по набору CheckPoint, зависит от свойства "shape", которое может принимать одно из трех значений:
- "Loop" - набор CheckPoint образует замкнутый контур. Этот контур делит пространство на части, внутри и снаружи контура.
- "Beam" - набор CheckPoint образует ломаную линию, крайние отрезки которой, при расчете деления пространства, продолжаются до другой разделяющей границы или до края.
- "Piece" - набор CheckPoint образует ломаную линию. Эта ломаная линия делит пространство только если отделяет часть, то есть пересекает другие разделяющие границы или край пространства.
Если часть разделяющие границы выходит за край пространства, то эта часть  не используется при делении пространства, но не пропадает из описания элемента. 

> [!note] 
> DividingBoundary имеет приоритет над DividingPlane. То есть когда одно пространство разделяется и разделяющей плоскостью и разделяющей границей, то сначала пространство разделяется вертикальными границами, а потом та часть, в которую попадает точка вставки разделяющей плоскости делится горизонтально на части.

CheckPoint (Контрольная точка) - точка определяющая узел сегмента и его свойства. CheckPoint должны быть в наборе, не менее двух для линейного элемента, например для незамкнутого DividingBoundary, или не менее трех для замкнутого, например для Border. Первая  контрольная точка задает начало и свойства первого отрезка, и последующих пока свойства не задаст другая контрольная точка.

Space (Пространство) - описывает объект который занимает предоставленный родительским элементом (строением или другим пространством) объем. Задается точкой вставки и набором разделяющих плоскостей и разделяющих границ. Пространство не имеет описания границ, его границы определяются граница и разделением родительского элемента, в той части где находится точка вставки. 

> [!note]
>  Для визуализации пространств можно использовать материалы с Space в названии в папке `Assets/Application/Materials`
>  ==в дальнейшем нужно будет реализовать генерацию материалов с палитрой из мягких цветов, которая используется в архитектуре для обозначения зон и помещений== 

Building (Строение) - часть здания включающая в себя пространства. Строение обладает всеми свойствами пространства, так же занимает объем, и может быть разделено на части, которые занимают другие пространства. Но в отличие от пространства строение обладает собственным краем описанным в Border, а не занимает заданный родителем объем.

Site (Строительная площадка) - объект задающий начальные координаты для всего документа и содержащий в себе все определения строений в проекте.

Material (Материал) - описание материала для геометрии элемента. Материалы описываются в проекте в отдельных файлах в папке `Material`
