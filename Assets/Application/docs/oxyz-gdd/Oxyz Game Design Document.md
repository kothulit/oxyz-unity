## 1. Назначение документа
Этот база знаний фиксирует идеи и текущее видение BIM-приложения на Unity, основанного на сущности [[Space]] , и служит для:
- реализации конкретных идей;
- планирования MVP и этапов разработки;
- принятия архитектурных решений;
- понимания, какие знания потребуются автору для реализации задуманного. 
Документ описывает прежде всего прототип-концепт, который должен подтвердить жизнеспособность ключевой идеи: модель здания строится не из набора слабо связанных объектов, а из иерархии [[Space]], внутри которых размещаются и адаптируются остальные элементы.
## 2. Видение продукта
### 2.1. Главная идея
Ключевая сущность приложения - [[Space]].
Вместо подхода, где модель состоит из множества отдельных слабо связанных элементов, приложение должно строить модель вокруг пространственной структуры здания:
- здание;
- корпуса;
- секции;
- этажи;
- квартиры;
- помещения;
- иные объемные зоны.
Все остальные элементы модели должны размещаться на основе [[Space]] и адаптироваться под изменения его геометрии.

### 2.2. Ценность приложения
Принятый подход в создании модели на основе [[Space]] и хранении модели с помощью [[XML представление|XML представления]] должно дать следующие преимущества:
- адаптивность модели к изменениям геометрии;
- быстрое моделирование объемов и основных архитектурных решений;
- автоматизированное формирование чертежей планов и разрезов;
- сокращение ручного труда при внесении изменений;
- прозрачное текстовое хранение модели;
- удобную интеграцию с внешними инструментами контроля версий;
- текстовый формат данных, обеспечивающий возможность доработки модели внешними приложениями;
- более точную принадлежность каждого элемента к части здания;
- структурированность и связанность модели на основе [[Space]] упрощают автоматизацию сложных задач;
- текстовый формат данных дает основу для будущей генерации вариантов модели через LLM;
- меньший объем файлов модели;
- читаемый формат хранения и возможность правки вручную.

### 2.3. Целевые пользователи
Приложение ориентировано на несколько ролей:
- архитектор: формирует архитектурную модель;
- конструктор: строит несущие элементы по границам [[Space]];
- инженер: добавляет источники и потребители, логически связывает их в системы и строит трассы сетей на основе этих связей;
- BIM-координатор: поддерживает библиотеку типов, координирует модель;
- главный инженер: анализирует решения и формирует замечания и предложения;
- заказчик: следит за изменениями в проекте, согласовывает решения;
- эксперт: проверяет принятые решения на соответствие требованиям.

### 2.5. Этапность развития продукта
==Финальная цель - платформа для всех стадий проектирования, но разработка начинается с прототипа-концепта.==
Приоритет прототипа:
- проверить, работает ли модель на основе Пространств;
- проверить, можно ли обеспечить адаптивность элементов;
- проверить, возможно ли и насколько практично XML-хранение проекта;
- проверить возможность автоматического формирования видов;
- убедиться, что модель данных пригодна для дальнейшего роста.
